\section{Requirements}

\pagenumbering{arabic}

\begin{itemize}
\item Provide only those features that Ada doesn't already provide, i.e. no wxThread as Ada provides tasking.
\item Simplify the interface to the types, the C++ methods are too complex; the Ada equivalents should be controlled types which call th
      default constructor on the object, then a create function which actually initialises it.s
\item Make the types look as Ada-like as possible, i.e. a thick binding.
\item Provide a method for internationalising applications, Ada doesn't provide a preprocessor so using \texttt{\_T()} or
      \texttt{\_()} isn't an option.
\item Don't use wxString, pass Ada String's to functions.
\item Try to match the objects to specific libraries, i.e. wxBase, wxCore, wxAdvanced, etc.
\item An event function needs to be implemented which will be called and handles the C -> Ada event mechanism.
\item When any object is passed to another, e.g. sizers, a list of items needs to be kept, so that nothing is leaked. This is a problem, because
      wxWidgets free's any items that are passed to it, i.e. a frame will delete it's children.
\item Exceptions, if there are any, need to be propogated to Ada.
\end{itemize}
